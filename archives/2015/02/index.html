parallel 的级别有methods,tests,classes,instances

  • classes：同一个class中用一个线程运行
  • instance: 同一个实例用同一个线程运行

pom.xml配置

引入testng 的jar包，和插件包

如果在terminal 中输入mvn test 在surefire插件的配置中如果没有配置configuration信息，maven会把src/test/java/ 下所有的类名中包含test的类进行测试，

如果想要执行自己写得testng.xml 可以右键点击该文件 点执行，就回执行配置的信息

intellij 多线程debug

2015 Feb 4

之所以要写这个，是因为之前在用eclipse的时候 设置的断点默认是可以多线程调试的，
换用intellij后我发现 只要一个线程断点停下之后，其他线程 根本就不会走，处于完全阻塞状态，
百度了好久也没有找到答案，结果还是在http://stackoverflow.com/上找到了解决办法，如下：


原来是intellij 设置的断点默认是all 也就是阻塞所有的线程，只需要把这里设置为Thread即可（这个对话框是右键点击断点出来的哦！）

这个估计是之前的老版本：
To quote the suspend policy

Item Description

+ All : When the breakpoint is hit, all threads are suspended
+ Thread : When the breakpoint is hit, the thread where the breakpoint is hit is     suspended.
+ None: No thread is suspended.
